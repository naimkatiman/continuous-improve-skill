# LinkedIn Post

I spent months watching AI coding agents fail in the exact same ways.

Skip research. Overbuild. Claim success without testing. Repeat the same mistakes.

So I built continuous-improvement — a tool that installs discipline into any AI coding agent with one command.

Here's how it works:

Every AI agent (Claude Code, Codex, Cursor, ChatGPT) has a config file where you define how it should behave. continuous-improvement injects a 7-rule operating loop:

Research → Plan → Execute (one thing) → Verify → Reflect → Learn → Iterate

That's it. No framework. No wrapper. No vendor lock-in.

Your agent. Same tools. Better output.

The newest version (v1.0) adds something I'm calling Mulahazah — Arabic for "observation." Your agent builds instincts over time. It watches what works, assigns confidence scores, and graduates from suggesting to auto-applying as patterns prove reliable.

Wrong instincts decay. Good ones compound.

The result: agents that actually get better the more you use them.

Open source. MIT licensed. Works across all major AI coding tools.

Try it: github.com/naimkatiman/continuous-improvement

If you've ever had an agent write 400 lines of code without checking what already exists — this is for you.
